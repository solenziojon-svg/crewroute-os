# Empire AI Knowledge System

## Three-Layer Architecture

**Raw Layer** — Unstructured field data (photos, job notes, chat logs, financial records, Telegram messages).  
**Wiki Layer** (this folder) — Synthesized principles, formulas, constraints, and decision frameworks.  
**Schema Layer** — Structured data models used by Supabase, agents, and applications.

## Usage Protocol (Mandatory)
1. Load the Empire Behavioral Prompt at the start of every session.
2. Read the relevant wiki pages before reasoning or generating code.
3. Update the Wiki layer when new validated patterns emerge from Raw data.
4. Push structured outputs into the Schema layer for long-term use.

## Self-Improving Loop
The system gets smarter every time a job is completed, a photo is analyzed, content is created, or a Magic Layer module is built — **provided** the knowledge is captured in the Wiki and Schema layers.
